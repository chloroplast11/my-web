# 个人品牌网站 · 需求文档（PRD）

> 版本 v0.3 · 状态：方案 + 视觉方向已确认，待开发
> 主角：**Chuck Chen**（前端工程师 / 摄影爱好者，现居东京）
> 用途：个人品牌展示，主要面向**面试 / 求职 / 对外介绍**场景。
> 网站语言：**英文**（UI 文案、首页内容、按钮等均为英文；博客文章本身可为中/英/日）。

## 1. 产品定位与目标

### 1.1 一句话定位
一个酷炫但克制、专业可信的个人网站，让访问者（主要是面试官/招聘方）在 1 分钟内了解"我是谁、我做过什么、我怎么想"，并能快速联系到我。

### 1.2 核心原则（North Star）
**网站本身就是作品集。** 访问者会同时评判内容和工程质量——加载速度、流畅度、移动端体验、代码质感与文案同等重要。因此本项目所有"酷炫"都服从一条底线：

> 动效不得拖慢首屏、不得卡顿、不得在手机上掉链子。

由此推导出三个模块的调性分层（与你最初的判断一致）：

| 模块 | 调性 | 动效强度 |
|---|---|---|
| 首页 Home | 高级、温润、编辑/纸质质感、有记忆点的动效（最初参考 godly.website 的暗色未来感，最终定为**浅色纸质编辑风**——见 1.4） | 高（但克制） |
| 博客 Blog | 简约、易读、专业 | 低 |
| 相册 Album | 主流摄影展示方案、性能优先 | 中（仅交互必需） |

### 1.3 衡量是否成功
- 面试官能在不滚动的情况下看到"我是谁 + 联系方式"。
- 首屏加载（LCP）< 2.5s，Lighthouse 性能/可访问性 ≥ 90。
- 移动端体验与桌面端一致流畅。

### 1.4 视觉与动效规范（已定稿，见原型 `theme-prototype-light.html`）

**主题：浅色纸质 / 编辑风（已确认，不做深/浅切换）**
经过暗色未来感 → 暖色暗调 → **浅色纸质**三轮迭代后定稿。整体观感像高级纸张/画廊印刷品：暖米底、深咖墨字、一抹暖陶土色点缀，配极淡颗粒质感。

| 项 | 值 |
|---|---|
| 背景 Paper | `#f3ede1` / 次级 `#ece4d4` |
| 文字 Ink | `#241e17`；次要 `#796f62`；更弱 `#a99d8c` |
| 强调 Accent | 暖陶土 `#9c6b3a`（仅小面积点缀） |
| 卡片 Surface | `#fbf6ec` / hover `#fffdf6` |

**字体（衬线主导的编辑感）**
- 标题/大字：**Newsreader**（衬线体，书卷/编辑气质，斜体用于金句）。
- 正文/标签：**Hanken Grotesk**（安静的无衬线体）。
- 刻意避开 Inter 等通用脸；不使用等宽字做大面积装饰（避免"科技感"）。

**入场动效（整体过场，已定稿）**
页面载入时三语问候 **「你好 → Hello → こんにちは」依次浮现并停留**（约 0.6s 间隔），随后整体一起淡出，接着遮罩像**相机光圈（circle/iris）从外向内收口**把页面"显影"出来，再触发首页标题逐行升起。
- 呼应主角的**摄影**身份（光圈）与**三语写作**身份（问候）。
- **重复访问用 `sessionStorage` 标记跳过**该过场（约 4s，每次都放会烦）；`prefers-reduced-motion` 下整段跳过。

**局部动效（已定稿，克制）**
标题逐行遮罩揭开、图片裁切擦入（clip-path）、数字滚动计数、滚动渐入、技术栈斜体跑马灯、导航滚动后收起加毛玻璃、链接下划线展开。原则：服从 1.2 的"不拖慢、不卡顿、不花哨"，全部受 `prefers-reduced-motion` 约束。

## 3. 技术方案选型

> 结论：基于你"自建后台、尽快上线、用现成编辑器、可接受少量月费"的诉求，推荐 **Next.js 全栈一体方案**——一个框架同时解决前台、后台 API 和图片优化，最快出活，又能在面试中体现全栈能力。

| 层 | 选型 | 理由 |
|---|---|---|
| 框架 | **Next.js（App Router）+ TypeScript** | 前台 + 后端 API/Server Actions 一体，`next/image` 天生解决相册图片优化与 SEO；部署最简单 |
| 样式 | **Tailwind CSS** | 出活快、便于维护设计系统 |
| 动效 | **Framer Motion**（滚动渐显/微交互）+ **Lenis**（平滑滚动）；首页英雄区可选 **GSAP** | 主流、性能可控、易加 `prefers-reduced-motion` 兜底 |
| 富文本编辑器 | **BlockNote**（首选）/ TipTap（备选） | Notion 风格、**开箱即用**：自带菜单、斜杠命令、拖拽排序、代码块、图片上传，原生 TypeScript，支持导出 HTML/Markdown，最省事 |
| 代码高亮 | **Shiki**（前台渲染）| 与 VS Code 同款主题，技术文档观感好 |
| 数据库 | **PostgreSQL**（Supabase 或 Neon 托管，含免费/低价档）+ **Prisma/Drizzle ORM** | 文章/标签/照片元数据存储，免运维 |
| 鉴权 | **Auth.js (NextAuth)**，单管理员账号 | 只有你一人登录后台，最简实现即可 |
| 图片存储与处理 | **Cloudinary**（首选，有免费额度，可少量付费）| 上传原图后**按需裁剪/压缩/转 AVIF·WebP**、自动 CDN 分发、自动生成缩略图，最适合"图很大"的相册 |
| 部署 | **Vercel**（前后端）+ Supabase/Neon（DB）+ Cloudinary（图） | 静态/Serverless 免运维，月费可控 |

> 备选说明（仅存档）：若未来想突出 Angular 能力，可改为 Angular(前台) + 独立 Node/NestJS(后台)，但需拆成两套、工作量更大，且 Notion 风格现成编辑器以 React 生态更成熟。**当前已确认采用 Next.js。**

预估月成本：起步阶段基本可控制在 **0–10 美元/月**（Vercel 免费档 + Supabase/Neon 免费或入门档 + Cloudinary 免费额度），图片量上来后按 Cloudinary 用量小幅增加。

> 待补：阿里职级英文译法（`Frontend Engineering Expert` vs `Staff Frontend Engineer`，待 Chuck 定）、真实邮箱与社交链接、阿里 5 个月的具体项目（可选）。

**动效要求**

**面试向硬性要求**
- 联系方式必须显眼（邮箱 / 社交链接）。
- 完整移动端适配。

### 4.2 博客 Blog

#### 4.2.1 前台（公开）

**列表页 `/blog`**
- 文章卡片：标题、摘要、**语言徽标（ZH / EN / JP）**、主题标签、日期、预计阅读时长、封面图（可选）。
- **过滤**：顶部按语言大分类切换（中 / 英 / 日 / 全部）+ 主题标签多选过滤（angular、react、journal…）。
- 站内搜索：列为 Phase 2（非首发必需）。

**详情页 `/blog/[slug]`**
- 正文渲染，**代码块语法高亮 + 一键复制**（技术文档刚需）。
- 元信息：语言、日期、阅读时长、标签。
- 目录 TOC（长文，可选）、上一篇/下一篇。
- 该页 `<html lang>` 按文章语言设置（zh / en / ja），利于 SEO 与无障碍。

#### 4.2.2 后台（鉴权保护）

- **登录**：单管理员账号，登录失败做基础限流。
- **文章管理 `/admin/posts`**：列表展示草稿/已发布，可筛选、搜索。
- **编辑器 `/admin/posts/new` `/edit/[id]`**：BlockNote（Notion 风格），支持标题、列表、引用、**代码块（含语言选择）**、表格、链接、图片（上传至 Cloudinary）。
- **文章元数据表单**：标题、slug（可自动从标题生成）、**语言（zh/en/ja 单选）**、标签（可新建/多选）、摘要、封面图、状态（草稿/已发布）、发布时间。
- **标签管理 `/admin/tags`**：标签增删改，含名称（可选颜色）。
- 操作：保存草稿 / 发布 / 下架 / 删除。

#### 4.2.3 语言模型（已确认）
- **每篇文章仅一种语言**，语言作为顶层大分类（不做同一篇的多语言翻译，不引入全站 i18n 翻译机制）。
- 文章数据含 `language` 字段（枚举：`zh` / `en` / `ja`）。

### 4.3 相册 Album

**核心矛盾**：原图可能很大、未来数量增长，需在"好看"与"性能"间取得平衡。采用市面主流摄影展示方案。

#### 4.3.1 前台（公开）`/photos`
- **布局**：横竖混排适配——首选 **masonry 瀑布流** 或 **justified rows（按行铺满，Flickr/Google Photos 风）**，避免纯网格裁切构图。
- **图片性能组合拳**：
  - 绝不直接加载原图；由 Cloudinary / `next/image` 按视口尺寸下发 **AVIF/WebP** 多尺寸（srcset）。
  - **懒加载** 视口外图片。
  - **模糊占位图（blurhash / LQIP）**：先出一团模糊色块再渐显，避免布局抖动（CLS）。
- **Lightbox 大图查看**：点击放大，键盘左右切换、移动端滑动；**此时才加载高清版**。
- **EXIF 元数据（已确认默认展示）**：在大图/详情处展示相机型号、镜头、光圈、快门、ISO、焦距等拍摄参数（摄影圈喜欢）。
- **分组（可选）**：相册/合集（Collections），可按合集或标签过滤。
- **虚拟滚动**：图片量很大时再做（Phase 2/3）。

#### 4.3.2 后台（鉴权保护）`/admin/photos`
- 上传照片至 Cloudinary，**自动生成缩略图 + blurhash**。
- 元数据：标题/说明、所属合集、标签、**自动提取 EXIF**。
- **隐私**：默认**剥离 GPS 定位信息**（防止照片暴露拍摄地点）。
- 排序、删除。

## 6. 非功能性需求

**性能（硬指标）**
- Core Web Vitals：LCP < 2.5s、CLS < 0.1、INP < 200ms。
- Lighthouse（移动端）性能 ≥ 90、可访问性 ≥ 90。
- 图片全量经 CDN + 现代格式；动画库懒加载；代码分割。

**可访问性（WCAG 2.1 AA 基线）**
- 全键盘可达、语义化 HTML、图片 alt、对比度达标、`prefers-reduced-motion` 兜底。

**SEO**
- 每页 meta + Open Graph 图；结构化数据（Person / Article）；`sitemap.xml`、`robots.txt`；RSS（可选，加分项）。

**响应式与兼容**
- 手机/平板/桌面全适配；目标浏览器为主流常青版本（Chrome/Safari/Firefox/Edge 近两版 + iOS Safari）。

**安全**
- 后台鉴权、登录限流、密钥走环境变量、HTTPS、编辑器输出内容净化。

**语言**
- UI/文案全英文；文章正文可中/英/日；文章页 `lang` 属性随文章语言设置。
